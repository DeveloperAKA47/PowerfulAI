## 一、开始开发前需要理解的基础概念

本次作业中的数学并不要求证明定理，也不要求手算复杂矩阵。开发时最重要的是理解：

1. 输入表示什么；
2. 数学公式中的每一步对应什么数组操作；
3. 数组形状怎样变化；
4. 为什么要处理除零、指数溢出和浮点误差。

### 1. 标量、向量、矩阵和高维数组

#### 标量

标量就是单个数字，例如：

```text
3
-1.5
0.8
```

NumPy 中的零维数组也可以表示标量：

```python
np.array(3.0)
```

它的形状是 `()`。

#### 向量

向量可以简单理解为一排数字：

```text
[3, 4]
```

形状是 `(2,)`。在 AI 应用中，一个向量常用来表示一个样本的特征，例如：

```text
[身高, 体重, 年龄]
```

也可以表示一句话经过模型处理后的 Embedding。

#### 矩阵

矩阵可以理解为多行向量组成的表格：

```text
[
    [1, 2, 3],
    [4, 5, 6],
]
```

形状是 `(2, 3)`，可以理解为：

```text
2 个样本；
每个样本有 3 个特征。
```

#### 更高维数组

例如：

```text
(batch_size, sequence_length, feature_dim)
```

可以表示：

```text
一批句子 × 每句话中的词 × 每个词的特征。
```

本次作业的大部分公式不仅适用于一维向量，也适用于多维数组。

---

### 2. 逐元素运算和矩阵乘法不是同一件事

假设：

```text
x = [1, 2]
y = [3, 4]
```

逐元素相乘：

```text
x * y = [1×3, 2×4] = [3, 8]
```

点积：

```text
x · y = 1×3 + 2×4 = 11
```

在 NumPy 中，`x * y` 是逐元素相乘；`np.sum(x * y)` 可以计算一维向量的点积。

矩阵乘法 `x @ y` 遵守“第一个矩阵的行乘以第二个矩阵的列”的规则。开发时一定要先确认接口需要的是逐元素运算，还是点积或矩阵乘法。

---

### 3. `axis` 表示沿哪个维度进行计算

假设数组：

```text
data = [
    [1, 2, 3],
    [4, 5, 6],
]
```

形状是 `(2, 3)`。

#### `axis=None`

对全部元素一起计算：

```text
1 + 2 + 3 + 4 + 5 + 6 = 21
```

#### `axis=0`

压缩第 0 维，也就是把各行合并，得到每一列的结果：

```text
[1+4, 2+5, 3+6] = [5, 7, 9]
```

结果形状为 `(3,)`。

#### `axis=1`

压缩第 1 维，也就是对每一行计算：

```text
[1+2+3, 4+5+6] = [6, 15]
```

结果形状为 `(2,)`。

#### `axis=-1`

表示最后一个维度。对于二维数组，`axis=-1` 等同于 `axis=1`。在 AI 数组中，最后一个维度经常表示“特征维度”，所以许多接口默认使用 `axis=-1`。

---

### 4. `keepdims=True` 为什么重要

假设 `x.shape == (2, 3)`。

```python
np.sum(x, axis=1)
```

结果形状为 `(2,)`。

```python
np.sum(x, axis=1, keepdims=True)
```

结果形状为 `(2, 1)`。

数值相同，但维度不同：

```text
不保留维度：
[6, 15]

保留维度：
[
    [6],
    [15],
]
```

当我们需要让每一行除以该行自己的范数时：

```text
原数组形状：   (2, 3)
每行范数形状： (2, 1)
```

NumPy 可以把 `(2, 1)` 广播到 `(2, 3)`，使每一行都除以自己的分母。因此在归一化、Softmax 等操作中，经常需要 `keepdims=True`。

---

### 5. 广播是什么

广播是 NumPy 自动扩展较小数组形状的规则。例如：

```text
矩阵形状：(2, 3)
向量形状：(3,)
```

执行相加时，向量会被看作同时加到矩阵的每一行。

广播不是随意扩展。两个维度从后往前比较时，必须满足以下任一条件：

```text
两个维度相等；
其中一个维度为 1；
其中一个数组缺少这个维度。
```

典型情况：

```text
(2, 3) 与 (3,)   可以广播
(2, 3) 与 (1, 3) 可以广播
(2, 3) 与 (2, 1) 可以广播
(2, 3) 与 (2,)   通常不能广播
```

---

### 6. 为什么要转换成浮点数组

整数数组适合保存类别编号，但许多 AI 数学运算会产生小数，例如：

```text
除法、平方根、指数、概率、平均值、归一化。
```

例如整数向量 `[3, 4]` 归一化后应为 `[0.6, 0.8]`，所以 `l2_norm`、`normalize`、`sigmoid`、`softmax` 和损失函数等，应使用浮点数组进行计算。

---

### 7. `eps` 是什么

`eps` 是一个很小的正数，例如：

```text
1e-7 = 0.0000001
```

它主要用于避免分母为 0。

零向量 `[0, 0]` 的长度为 0，直接归一化会计算：

```text
[0, 0] / 0
```

结果可能出现 `NaN`。因此可以把分母最低限制为 `eps`：

```text
安全分母 = max(真实分母, eps)
```

这并不是说零向量真的有 `eps` 这么长，而是为了让计算机安全完成除法。

## 二、`one_hot()`：独热编码

接口：

```python
def one_hot(
    indices: ArrayLike,
    num_classes: int,
    *,
    dtype: Any | None = None,
) -> np.ndarray:
    ...
```

### 1. 它解决什么问题

机器学习模型通常不能直接把类别名称作为数值输入。例如颜色类别：

```text
红色、绿色、蓝色
```

可以先编号：

```text
红色 → 0
绿色 → 1
蓝色 → 2
```

但直接使用数字可能造成错误的大小含义。独热编码把每个类别转换成一个只有一个位置为 1、其他位置为 0 的向量：

```text
类别 0 → [1, 0, 0]
类别 1 → [0, 1, 0]
类别 2 → [0, 0, 1]
```

这三个向量地位平等，不包含人为的大小顺序。

### 2. 数学理解

假设类别总数为 `C`，类别编号为 `k`，且：

```text
0 ≤ k < C
```

独热向量长度为 `C`，第 `k` 个位置为 1：

```text
one_hot(k)[j] =
    1，j = k
    0，j ≠ k
```

例如 `C=4，k=2`，结果为 `[0, 0, 1, 0]`。

### 3. 多个类别编号时的形状

输入形状与输出形状关系：

```text
输出形状 = indices.shape + (num_classes,)
```

例如：

```text
输入形状 ()      → 输出形状 (C,)
输入形状 (N,)    → 输出形状 (N, C)
输入形状 (A, B)  → 输出形状 (A, B, C)
```

最后新增的维度表示类别。

### 4. 为什么类别编号必须是整数

独热编码需要把类别编号当作数组下标。合法编号是 `0、1、2` 等整数；不合法的输入包括 `1.5`、字符串、负数，以及大于等于 `num_classes` 的编号。

### 5. 数学到 NumPy 的映射

实现思路可以分成：

```text
第一步：创建全 0 数组；
第二步：确定每个样本对应的行位置；
第三步：把类别编号对应的位置改为 1。
```

关键不是循环逐个写，而是使用高级索引一次性修改多个位置。对于多维输入，需要理解：输出最后一个维度才是类别维度，写入前可以把前面的样本维度临时展平。

### 6. 最低必要知识

```text
独热编码是把类别编号变成单位向量；
向量长度等于类别总数；
类别编号决定 1 所在的位置；
输出比输入多一个类别维度。
```

## 三、`l2_norm()`：L2 范数

接口：

```python
def l2_norm(
    value: ArrayLike,
    *,
    axis: AxisLike = None,
    keepdims: bool = False,
) -> np.ndarray:
    ...
```

### 1. L2 范数是什么

二维坐标中的点 `(3, 4)` 到原点的距离是：

```text
sqrt(3² + 4²) = 5
```

所以向量 `[3, 4]` 的 L2 范数是 5。

一般公式：

```text
||x||₂ = sqrt(x₁² + x₂² + ... + xₙ²)
```

实现思路是：

```text
逐元素平方；
沿指定 axis 求和；
对结果开平方。
```

### 2. 为什么平方后再开平方

平方会消除正负号，因为长度不应为负；同时它对应欧氏几何中的直线距离。

### 3. 不同 `axis` 的含义

输入：

```text
[
    [3, 4],
    [0, 5],
]
```

- `axis=None`：把全部元素看成一个长向量，结果是 `sqrt(50)`。
- `axis=1`：按行计算，结果是 `[5, 5]`。
- `axis=0`：按列计算，结果是 `[3, sqrt(41)]`。

如果 `axis=1, keepdims=True`，结果形状是 `(2, 1)`，便于后续广播。

### 4. L2 范数的性质

```text
||x||₂ ≥ 0；
只有零向量的范数为 0；
所有元素乘以常数 a，长度会乘以 |a|。
```

### 5. 数学到 NumPy 的映射

核心就是：

```text
sqrt(sum(x * x))
```

不需要逐元素 Python 循环。

## 四、`normalize()`：L2 归一化

接口：

```python
def normalize(
    value: ArrayLike,
    *,
    axis: AxisLike = -1,
    eps: float | None = None,
) -> np.ndarray:
    ...
```

### 1. 归一化做了什么

归一化是把向量缩放成长度为 1，同时保持方向不变：

```text
normalize(x) = x / ||x||₂
```

例如：

```text
x = [3, 4]
||x||₂ = 5
normalize(x) = [0.6, 0.8]
```

新向量长度是 1。

### 2. 为什么方向不变

向量所有分量都除以同一个正数，分量比例不变：

```text
3 : 4 = 0.6 : 0.8
```

所以只改变长度，不改变方向。

### 3. 多行矩阵的归一化

输入：

```text
[
    [3, 4],
    [5, 12],
]
```

使用 `axis=-1` 时，对每一行分别归一化：

```text
[
    [3/5, 4/5],
    [5/13, 12/13],
]
```

每一行归一化后的长度都约为 1。

### 4. 为什么需要 `keepdims=True`

原数组形状为 `(2, 2)`。每行范数保留成 `(2, 1)`，可以明确表示“每一行有一个分母”，然后通过广播让每一行所有元素除以自己的范数。

### 5. 零向量和 `eps`

零向量的范数是 0。工程中使用：

```text
安全分母 = max(范数, eps)
```

于是零向量除以 `eps` 后仍然是零向量，避免 `NaN`。

### 6. 在 AI 中的作用

归一化可以消除向量长度差异，让后续比较更关注方向或相对结构，而不是谁的数值整体更大。

## 五、`cosine_similarity()`：余弦相似度

接口：

```python
def cosine_similarity(
    x: ArrayLike,
    y: ArrayLike,
    *,
    axis: int = -1,
    eps: float | None = None,
) -> np.ndarray:
    ...
```

### 1. 它衡量什么

余弦相似度衡量两个向量的方向是否接近，而不是长度是否接近：

```text
cosine(x, y) =
(x · y) / (||x||₂ × ||y||₂)
```

### 2. 点积是什么

```text
x · y = x₁y₁ + x₂y₂ + ... + xₙyₙ
```

NumPy 思路是先逐元素相乘，再沿特征轴求和。

### 3. 为什么能表示方向

几何关系：

```text
x · y = ||x||₂ × ||y||₂ × cos(θ)
```

所以：

```text
方向相同：约为 1；
互相垂直：约为 0；
方向相反：约为 -1。
```

### 4. 为什么要除以范数

只使用点积会受到向量长度影响。除以两个向量长度后，消除了整体缩放的影响。例如 `[1,0]` 与 `[1,0]`、`[100,0]` 与 `[100,0]` 的相似度都应为 1。

### 5. 零向量问题

零向量没有方向，严格数学上余弦相似度没有定义。工程中使用 `eps` 避免除零，通常得到 0。要理解这是工程约定。

### 6. 广播

如果：

```text
x.shape = (5, 3)
y.shape = (3,)
```

可以理解为计算 5 个三维向量分别与同一个三维向量的相似度，结果形状是 `(5,)`。

如果输入不能广播，就无法建立对应关系，应抛出 `ShapeError`。

## 六、`pairwise_squared_distance()`：两组向量的两两平方距离

接口：

```python
def pairwise_squared_distance(
    x: ArrayLike,
    y: ArrayLike,
) -> np.ndarray:
    ...
```

### 1. 输入和输出

```text
x.shape = (m, d)
y.shape = (n, d)
```

含义：

```text
x 有 m 个样本；
y 有 n 个样本；
每个样本有 d 个特征。
```

输出形状是 `(m, n)`，其中 `result[i, j]` 表示 `x[i]` 与 `y[j]` 的平方距离。

### 2. 平方欧氏距离

普通欧氏距离：

```text
distance(x, y) =
sqrt((x₁-y₁)² + ... + (x_d-y_d)²)
```

平方距离去掉最外层平方根：

```text
distance²(x, y) =
(x₁-y₁)² + ... + (x_d-y_d)²
```

很多算法只需要比较距离大小，去掉平方根不会改变排序，同时更省计算。

### 3. 关键恒等式

```text
||x-y||²
= (x-y)·(x-y)
= x·x - 2x·y + y·y
= ||x||² + ||y||² - 2(x·y)
```

这是本接口最重要的数学公式。

### 4. 扩展到两组向量

- `x` 每行平方和：形状 `(m, 1)`；
- `y` 每行平方和再转置：形状 `(1, n)`；
- `x @ y.T`：形状 `(m, n)`，包含所有向量对的点积；
- 三者通过广播得到 `(m, n)` 的距离矩阵。

形状关系：

```text
(m, 1) + (1, n) - 2 × (m, n) → (m, n)
```

### 5. 例子

```text
x = [[0,0], [1,0]]
y = [[0,0], [2,0]]
```

结果：

```text
[
    [0, 4],
    [1, 1],
]
```

### 6. 为什么最后限制不小于 0

数学上平方距离不可能为负数，但浮点舍入可能得到 `-1e-15`。这是误差，应把结果最低限制为 0。

## 七、`relu()`：ReLU 激活函数

接口：

```python
def relu(value: ArrayLike) -> np.ndarray:
    ...
```

### 1. 数学定义

```text
ReLU(x) =
    x，x ≥ 0
    0，x < 0
```

也就是：

```text
ReLU(x) = max(x, 0)
```

### 2. 例子

```text
输入：[-3, -1, 0, 2, 5]
输出：[0, 0, 0, 2, 5]
```

### 3. 数组含义

ReLU 对每个元素独立处理，不改变数组形状。

### 4. 为什么用于神经网络

它引入非线性，使多层网络不能简单合并成一次线性运算。对应用开发者，只需理解：

```text
保留正信号；
抑制负信号；
逐元素计算；
形状不变。
```

## 八、`leaky_relu()`：带负区间斜率的 ReLU

接口：

```python
def leaky_relu(
    value: ArrayLike,
    negative_slope: float = 0.01,
) -> np.ndarray:
    ...
```

### 1. 数学定义

```text
LeakyReLU(x) =
    x，x ≥ 0
    αx，x < 0
```

其中 `α = negative_slope`。

### 2. 例子

当 `negative_slope=0.1`：

```text
输入：[-3, -1, 0, 2]
输出：[-0.3, -0.1, 0, 2]
```

### 3. 与普通 ReLU 的区别

普通 ReLU 把所有负数变成 0；Leaky ReLU 让负数保留一个较小的负值。

### 4. 参数限制

`negative_slope` 应为非负数。等于 0 时退化为普通 ReLU；负数会偏离本接口的含义，应拒绝。

### 5. 数学到 NumPy 的映射

这是条件选择：

```text
x >= 0 时选择 x；
否则选择 negative_slope * x。
```

应使用数组条件向量化处理。

## 九、`sigmoid()`：Sigmoid 激活函数

接口：

```python
def sigmoid(value: ArrayLike) -> np.ndarray:
    ...
```

### 1. 数学公式

```text
sigmoid(x) = 1 / (1 + exp(-x))
```

### 2. 输出范围

Sigmoid 把任意实数映射到 0 和 1 之间：

```text
很小的负数 → 接近 0；
0           → 0.5；
很大的正数 → 接近 1。
```

典型值：

```text
sigmoid(-2) ≈ 0.119
sigmoid(0)  = 0.5
sigmoid(2)  ≈ 0.881
```

### 3. 对称性质

```text
sigmoid(-x) = 1 - sigmoid(x)
```

### 4. 直接公式为什么可能溢出

当 `x=-1000`：

```text
exp(-x) = exp(1000)
```

可能超过浮点数范围，变成 `inf`。

### 5. 数值稳定的分段公式

当 `x >= 0`：

```text
1 / (1 + exp(-x))
```

当 `x < 0`：

```text
exp(x) / (1 + exp(x))
```

负区间公式与原公式等价，因为原公式分子分母同时乘以 `exp(x)` 即可得到它。分段后，指数的输入不会成为很大的正数，从而避免溢出。

### 6. 数学到 NumPy 的映射

```text
建立 x >= 0 的布尔掩码；
正区间使用原公式；
负区间使用等价公式；
把两部分写入同形状结果。
```

## 十、`softmax()`：把一组分数转换为概率分布

接口：

```python
def softmax(
    value: ArrayLike,
    *,
    axis: int = -1,
) -> np.ndarray:
    ...
```

### 1. 输入是什么

Softmax 输入通常叫 logits，是模型对各类别给出的原始分数。它们可以为正、为负，不要求总和为 1，不能直接当作概率。

### 2. 数学公式

```text
softmax(xᵢ) =
exp(xᵢ) / Σ exp(xⱼ)
```

每个分数先取指数，再除以全部指数值之和。

### 3. 为什么结果总和为 1

所有输出使用同一个分母：

```text
S = exp(x₁) + ... + exp(xₙ)
```

相加后得到 `S/S=1`。指数值总是正数，所以每个结果也为正。

### 4. 保持大小关系

如果 `x₁ > x₂`，则 `exp(x₁) > exp(x₂)`，所以 Softmax 后仍然是第一个结果更大。

### 5. `axis` 的意义

如果 `logits.shape == (batch_size, num_classes)`，通常沿最后一个类别维度计算，即 `axis=-1`。这样每个样本的类别概率之和为 1。

错误地使用 `axis=0`，会在不同样本之间归一化。

### 6. 指数溢出

输入 `[1000,1001,1002]` 时，直接取指数可能全部成为 `inf`，最后出现 `inf/inf=NaN`。

### 7. 减去最大值为什么不改变结果

Softmax 对整体平移不敏感：

```text
softmax(x) = softmax(x-c)
```

因为：

```text
exp(xᵢ-c) = exp(xᵢ) × exp(-c)
```

分子和分母中的公共因子会约掉。

选择 `c=max(x)`：

```text
[1000,1001,1002] - 1002 = [-2,-1,0]
```

最大指数变成 `exp(0)=1`，不会溢出。

### 8. 为什么要 `keepdims=True`

每组最大值和指数和都应保留被压缩的维度，使每一行能够减去自己的最大值、除以自己的总和。

### 9. 零维数组

零维数组只有一个标量，没有“一组类别分数”这一维，因此接口规定抛出 `ShapeError`。

## 十一、`mse_loss()`：均方误差损失

接口：

```python
def mse_loss(
    prediction: ArrayLike,
    target: ArrayLike,
    *,
    reduction: Reduction = "mean",
) -> np.ndarray | float:
    ...
```

### 1. 损失函数是什么

损失函数衡量模型预测值与真实目标之间差了多少。损失越小，通常说明预测越接近目标。

### 2. 逐元素公式

```text
error = prediction - target
loss = error²
```

例子：

```text
prediction = [1, 3]
target     = [2, 1]
error      = [-1, 2]
loss       = [1, 4]
```

### 3. 为什么平方

平方可以消除正负抵消，并对大误差给予更重惩罚：

```text
误差 1  → 损失 1
误差 2  → 损失 4
误差 10 → 损失 100
```

### 4. `reduction`

逐元素损失 `[1,4]`：

```text
"none" → [1,4]
"sum"  → 5
"mean" → 2.5
```

`"none"` 返回广播后的逐元素损失；`"sum"` 和 `"mean"` 返回单个数。

### 5. 广播

`prediction` 与 `target` 可以按照 NumPy 规则广播。例如 `(4,3)` 与 `(3,)` 可以逐元素比较。无法广播时，就没有有效的对应关系，应把 NumPy 的 `ValueError` 转成 `ShapeError`。

### 6. 空数组

空数组没有普通意义上的平均值。NumPy 直接求平均可能产生 `NaN` 和警告。本作业约定可将空损失数组的 `"mean"` 处理为 `0.0`。

### 7. 适用理解

MSE 常用于连续数值预测，并且对大误差比较敏感。

## 十二、`mae_loss()`：平均绝对误差损失

接口：

```python
def mae_loss(
    prediction: ArrayLike,
    target: ArrayLike,
    *,
    reduction: Reduction = "mean",
) -> np.ndarray | float:
    ...
```

### 1. 逐元素公式

```text
error = prediction - target
loss = abs(error)
```

例子：

```text
prediction = [1, 3]
target     = [2, 1]
error      = [-1, 2]
loss       = [1, 2]
```

### 2. 为什么取绝对值

绝对值消除正负号，预测偏高 3 和偏低 3 的错误程度相同，也不会发生正负抵消。

### 3. 与 MSE 的区别

误差为 `1、2、10` 时：

```text
MAE：1、2、10
MSE：1、4、100
```

因此：

```text
MSE 对大误差更敏感；
MAE 对极端误差相对更稳健。
```

### 4. `reduction`

规则与 MSE 完全相同：

```text
"none" → 返回逐元素绝对误差；
"sum"  → 全部相加；
"mean" → 求平均。
```

所以两个损失函数应复用相同的归约校验和处理逻辑。

### 5. 一个不阻碍开发的微积分补充

绝对值函数在 0 点有尖角，从严格微积分角度看该点没有普通导数。但本次作业只实现前向损失值，不实现反向传播，所以不需要进一步研究。

## 十三、11 个 API 之间的关系

### 1. 编码

```text
one_hot
```

把类别编号转换成模型容易处理的数组表示。

### 2. 向量长度和比较

```text
l2_norm
normalize
cosine_similarity
pairwise_squared_distance
```

关系：

```text
l2_norm：计算长度；
normalize：除以长度；
cosine_similarity：比较方向；
pairwise_squared_distance：比较位置距离。
```

### 3. 激活函数

```text
relu
leaky_relu
sigmoid
softmax
```

关系：

```text
ReLU / Leaky ReLU：逐元素非线性变换；
Sigmoid：把单个分数压到 0～1；
Softmax：把一组类别分数变成总和为 1 的分布。
```

### 4. 损失函数

```text
mse_loss
mae_loss
```

关系：

```text
MSE：平方误差，对大错误惩罚更重；
MAE：绝对误差，对极端错误相对稳健。
```

## 十四、开发时的统一思考顺序

实现每个 API 前，建议先回答以下问题。

### 1. 输入是什么数学对象

```text
类别编号？
一个向量？
一批向量？
一组类别分数？
预测值和目标值？
```

### 2. 输出形状是什么

```text
one_hot：多出类别维度；
l2_norm：压缩指定轴；
normalize：形状不变；
cosine_similarity：压缩特征轴；
pairwise_squared_distance：(m,d) 与 (n,d) 得到 (m,n)；
激活函数：形状不变；
损失函数 none：广播后的形状；
损失函数 sum/mean：单个浮点数。
```

### 3. 运算是逐元素还是矩阵运算

```text
ReLU：逐元素；
Sigmoid：逐元素；
MSE：逐元素；
余弦相似度：逐元素乘法后求和；
两两距离：矩阵乘法；
Softmax：逐元素指数加沿轴求和。
```

### 4. 是否需要浮点计算

包含以下操作时通常要转换成浮点：

```text
除法、平方根、指数、平均值、概率、归一化。
```

### 5. 是否存在数值风险

```text
分母可能为 0 → 使用 eps；
指数可能溢出 → 使用稳定公式；
理论非负结果出现微小负数 → 限制最低为 0；
浮点结果比较 → 使用 np.allclose。
```

### 6. 参数和形状是否合法

```text
类型错误 → TypeError；
参数取值错误 → ValueError；
数组形状错误 → ShapeError。
```

## 十五、最终数学速查表

| API | 核心数学含义 | 输出形状变化 | 主要数值问题 |
|---|---|---|---|
| `one_hot` | 类别编号变成单位向量 | 最后增加类别维度 | 类别越界 |
| `l2_norm` | 平方、求和、开平方 | 压缩指定轴 | 整数应转浮点 |
| `normalize` | 向量除以自身长度 | 不变 | 除以 0 |
| `cosine_similarity` | 点积除以两个向量长度 | 压缩特征轴 | 零向量、广播 |
| `pairwise_squared_distance` | 两组向量的两两平方距离 | `(m,d),(n,d)→(m,n)` | 浮点误差产生小负数 |
| `relu` | `max(x,0)` | 不变 | 无明显风险 |
| `leaky_relu` | 负区间乘小斜率 | 不变 | 斜率参数非法 |
| `sigmoid` | 映射到 0～1 | 不变 | 指数溢出 |
| `softmax` | 一组分数变成和为 1 的分布 | 不变 | 指数溢出、axis |
| `mse_loss` | 平方误差 | 由 reduction 决定 | 广播、空数组 |
| `mae_loss` | 绝对误差 | 由 reduction 决定 | 广播、空数组 |
